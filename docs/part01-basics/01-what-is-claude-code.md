# 第 1 课：什么是 Claude Code？

## 学习目标

完成本课后，你将能够：

1. 理解什么是 AI 编程助手，以及它和传统软件工具的区别
2. 说出 Claude Code 的核心能力（编辑文件、运行命令、搜索代码、协调工作流）
3. 区分 Claude Code 与 ChatGPT、GitHub Copilot 等工具的差异
4. 理解学习 Claude Code 源码的价值
5. 了解 Claude Code 使用的技术栈全貌

---

## 1.1 什么是 AI 编程助手？

### 用一个生活类比来理解

想象你正在装修一栋房子。传统的做法是：你自己拿着锤子、钉子，一砖一瓦地搭建。而 AI 编程助手就像是一位经验丰富的"工程师搭档"——你告诉他"帮我把卧室的墙刷成浅蓝色"，他就会自己找到正确的墙壁、选好涂料、完成施工，甚至告诉你"我顺便把旁边一个小裂缝也补了"。

在软件开发的世界里：

| 概念 | 类比 |
|------|------|
| **你的代码项目** | 你正在装修的房子 |
| **代码文件** | 房子里的各个房间 |
| **AI 编程助手** | 你的工程师搭档 |
| **你的自然语言指令** | 你对工程师说的话 |

你不再需要记住每一行代码怎么写——你只需要用自然语言描述你想做什么，AI 助手就能帮你完成。

### AI 编程助手的三种"等级"

1. **代码补全工具**（像"自动填词"）：在你打字时预测下一行代码，比如 GitHub Copilot 的行内补全
2. **对话式编程助手**（像"编程顾问"）：你可以和它聊天、问问题，它给你代码片段，比如 ChatGPT
3. **自主编程代理**（像"编程搭档"）：它不仅能给建议，还能**直接在你的电脑上操作**——编辑文件、运行命令、搜索代码。**Claude Code 就是这个等级**

## 1.2 Claude Code 具体能做什么？

Claude Code 是 Anthropic 公司开发的一款**命令行 AI 编程代理**。"命令行"意味着它运行在终端（Terminal）中，而不是像普通 App 那样有图形界面。

### 核心能力

#### 🔧 编辑文件
Claude Code 可以直接读取和修改你电脑上的代码文件。它不只是"建议你改什么"，而是**直接帮你改好**。

```
你说：帮我把 login 函数里的错误提示改成中文
Claude Code：好的，我来修改 src/auth/login.ts 文件...（直接修改完成）
```

#### 💻 运行命令
它可以在你的终端中执行命令，比如安装依赖、运行测试、启动服务器：

```
你说：运行一下测试看看有没有问题
Claude Code：我来执行 npm test...（运行命令并分析结果）
```

#### 🔍 搜索代码
面对一个庞大的代码库（成千上万个文件），Claude Code 可以快速搜索和定位相关代码：

```
你说：找一下项目里所有处理用户认证的代码
Claude Code：我搜索了相关文件，找到以下 15 个相关文件...
```

#### 🤖 协调工作流
Claude Code 可以编排复杂的多步骤任务——先分析问题、再修改代码、然后运行测试验证、最后提交 Git：

```
你说：修复这个 bug，然后提交一个 PR
Claude Code：让我先理解这个 bug...修改代码...运行测试确认修复...创建 Git 提交...创建 PR
```

## 1.3 Claude Code 与其他工具的区别

| 特性 | ChatGPT | GitHub Copilot | Claude Code |
|------|---------|----------------|-------------|
| **交互方式** | 网页对话 | 编辑器内嵌 | 命令行终端 |
| **能否操作文件** | ❌ 不能 | ⚠️ 有限（建议修改） | ✅ 直接读写文件 |
| **能否运行命令** | ❌ 不能 | ❌ 不能 | ✅ 执行终端命令 |
| **能否搜索代码** | ❌ 不能 | ⚠️ 有限 | ✅ 全项目搜索 |
| **工作模式** | 一问一答 | 行内补全 | 自主代理（Agent） |
| **上下文范围** | 对话窗口 | 当前文件 | 整个项目 |
| **适合场景** | 学习、问答 | 日常编码加速 | 复杂工程任务 |

**关键区别：Claude Code 是一个"Agent"（代理）**

所谓"Agent"，就是它能够自主地、多步骤地完成任务。普通的 AI 对话就像是"你问一句、它答一句"；而 Claude Code 是"你布置一个任务，它自己规划步骤、自己执行、遇到问题自己调整"。

就好比：
- ChatGPT = 你打电话问一位专家建议
- GitHub Copilot = 专家在你身边看你写字，随时给你递一支更好的笔
- Claude Code = 专家直接坐到你的电脑前，帮你干活

## 1.4 为什么学习 Claude Code 的源码？

你可能会问：我直接用 Claude Code 不就行了，为什么要学它的源码？

### 原因 1：理解"AI 编程代理"的工作原理

Claude Code 是目前最复杂的开源 AI 编程代理之一。学习它的源码，你会理解：
- AI 如何与人类协作？
- 一个 AI 代理系统是怎么设计的？
- 工具调用（Tool Use）机制是什么？
- 权限控制如何实现？

### 原因 2：学习工业级 TypeScript 项目

Claude Code 拥有约 **1900 个源码文件、51 万行代码**。这是一个真实的、高质量的大型项目。通过阅读它，你将学到：
- 大型项目如何组织目录结构
- TypeScript 类型系统的高级用法
- React 组件化设计模式
- 异步编程的最佳实践

### 原因 3：掌握前沿技术栈

Claude Code 使用的技术栈代表了 2024-2025 年前端/全栈开发的前沿：

| 技术 | 用途 |
|------|------|
| **TypeScript** | 主编程语言，提供类型安全 |
| **Bun** | 超快的 JavaScript 运行时，替代 Node.js |
| **React + Ink** | 在终端中渲染 UI（是的，React 不只是做网页！） |
| **Commander.js** | 命令行参数解析 |
| **Zod v4** | 运行时数据验证和 Schema 定义 |
| **ripgrep** | 超快的代码搜索引擎 |
| **MCP SDK** | Model Context Protocol，AI 与外部工具的通信协议 |
| **Anthropic SDK** | 与 Claude AI 模型通信 |
| **OpenTelemetry** | 可观测性和遥测 |
| **GrowthBook** | 特性开关（Feature Flag）管理 |
| **OAuth/JWT** | 认证和授权 |

### 原因 4：培养"读代码"的能力

在真实的软件开发工作中，**阅读代码的时间远多于编写代码**。学习阅读一个大型开源项目，是成为优秀程序员的必经之路。

## 动手练习

### 练习 1：感受 Claude Code 的项目规模

打开终端，输入以下命令，查看 Claude Code 源码的基本信息：

```bash
# 进入项目目录
cd /path/to/claude-code-main

# 查看 src 目录下有多少个子目录
ls src/

# 统计源码文件总数
find src -type f | wc -l

# 统计代码总行数
find src -type f -exec cat {} + | wc -l
```

### 练习 2：浏览核心文件

用任意文本编辑器打开以下文件，先大概浏览一下（不需要看懂！）：

```bash
# 项目入口文件
code src/main.tsx

# 工具类型定义
code src/Tool.ts

# 工具注册
code src/tools.ts
```

思考题：
- `main.tsx` 文件的后缀是 `.tsx`，你觉得这意味着什么？
- `Tool.ts` 文件中你能看到多少个 `type` 或 `interface` 关键字？
- `tools.ts` 中导入了多少个不同的工具？

### 练习 3：对比思考

打开 ChatGPT（chat.openai.com）或者任何你用过的 AI 工具，问它一个编程问题。然后思考：
- 这个 AI 工具能帮你直接修改电脑上的文件吗？
- 它能运行命令吗？
- 它了解你整个项目的上下文吗？
- Claude Code 在这些方面有什么不同？
