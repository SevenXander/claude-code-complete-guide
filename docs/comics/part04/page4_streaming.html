第4话: 流式响应 Streaming
4/8
SCENE 1 - 为什么要"流式"？
为什么不等 LLM 全部说完再显示？
想象你在餐厅点菜：
❌ 等所有菜做完一起上 = 干等10分钟
✅ 做好一道上一道 = 马上能吃！
流式 = 边生成边显示
SCENE 2 - 数据流长什么样？
Claude API → QueryEngine 的数据流:
chunk 1 → 🧠 thinking: "让我分析一下..."
chunk 2 → 📝 text: "我来帮你修改..."
chunk 3 → 📝 text: "这个文件的代码"
chunk 4 → 🔧 tool_use: Edit("app.ts", ...)
每个 chunk 到达后立即 yield 给调用者显示！不等待！
SCENE 3 - 普通模式 vs 流式执行
❌ 普通模式
等待
等流式完成
执行
执行工具
总计 5s
✅ 流式执行
检测
检测
并行
边流式边执行
总计 3s 🚀
StreamingToolExecutor 让工具 边流式边执行！
SCENE 4 - 关键代码
// 流式接收 + 并行执行 for await (const message of callModel({ messages, systemPrompt, tools })) { // 检测到工具调用 → 立即提交执行 if (msg.type === 'tool_use') streamingExecutor.submit(msg) yield message // 立即返回! }
